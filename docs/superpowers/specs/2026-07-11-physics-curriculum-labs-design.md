# 物理之美人教版初中仿真实验设计

## 1. 目标

将“物理之美”从知识主题演示目录升级为可用于课堂和自主学习的人教版初中物理仿真实验平台。平台覆盖用户提供清单中的 65 个实验，允许学生通过拖动、连接、挂接、调节、读数和记录等操作完成实验，并保存实验数据、自动计算结果和生成实验报告。

本设计采用用户确认的方案 1：在现有 React 应用中建立原生、可复用的仿真实验框架，并为不同实验配置独立的物理控制器。现有 133 个物理实验保留，作为“拓展实验”独立展示。

## 2. 权威范围与版本口径

## 3. 信息架构

### 3.1 物理首页

物理首页设置两个一级标签：

- 教材实验：默认标签，展示 65 项人教版实验。
- 拓展实验：保留现有 133 个主题实验和现有详情路由。

教材实验区采用与参考图一一致的目录式布局：

- 左侧目录树：教材版本、册别、章节。
- 顶部筛选：全部、必做、选做；实验类型；关键词搜索。
- 排序：教材顺序、最近使用、名称。
- 卡片信息：真实实验装置缩略图、实验名称、章节、必做/选做标签、完成状态。
- 统计信息：当前筛选结果数量，不用营销式大数字卡片占据首屏。

教材目录固定为：

```text
人教版
├── 八年级上册（第 1-6 章）
├── 八年级下册（第 7-12 章）
└── 九年级全一册（第 13-22 章）
```

最终状态下，65 项教材实验全部可进入且可完整操作。开发期间尚未完成的实验可显示“制作中”，但不得用静态占位页冒充完成实验。

### 3.2 实验详情与实验台

实验详情由两个连续区域组成：

1. 浅色实验信息区：实验名称、册别、章节、类型，以及目的、原理、器材、步骤、结论、补充六项内容。
2. 深色仿真实验台：参考用户提供的浮力 HTML 和图二，提供器材、步骤、实时读数、数据表格与实验报告。

实验台桌面端采用三栏结构：左侧器材区、中间实验场景、右侧步骤和当前读数。窄屏改为场景优先布局，器材和步骤通过抽屉打开，保证主要操作区域不被遮挡。

## 4. 技术架构

### 4.1 分层结构

教材实验不再嵌入旧站点 iframe，而是在主 React 应用中原生实现：

```text
教材目录数据
    ↓
实验定义（内容、器材、步骤、测量字段）
    ↓
通用实验运行时（拖拽、吸附、步骤、记录、撤销、报告）
    ↓
实验专用控制器（热学、电路、力学、光学、电磁学）
    ↓
SVG / Canvas 实验场景
```

边界如下：

- `curriculum`：只负责教材、册别、章节、类型和实验内容。
- `runtime`：只负责通用交互、会话、步骤推进、数据记录、撤销和重置。
- `apparatus`：只负责器材视觉、连接点、碰撞区域和读数显示。
- `labs`：每个实验独立定义操作规则、物理计算和完成条件。
- `sessions`：只负责本地持久化、实验记录和报告。

### 4.2 实验定义

每个实验使用强类型定义，至少包含：

```ts
interface TextbookPhysicsExperiment {
  id: string
  title: string
  textbook: '人教版'
  volume: '八年级上册' | '八年级下册' | '九年级全一册'
  chapter: string
  requirement: '必做' | '选做'
  activityType: '基础技能' | '测量' | '探究' | '演示' | '模型'
  purpose: string[]
  principle: string[]
  apparatus: ApparatusDefinition[]
  steps: ExperimentStepDefinition[]
  conclusion: string[]
  supplement: string[]
  measurements: MeasurementDefinition[]
  controller: ExperimentControllerId
}
```

工作簿中的“建议必做”归入“必做”，原始类型文字保留在实验补充中，避免筛选口径过细。

### 4.3 通用实验运行时

通用运行时提供：

- 指针事件统一处理，支持鼠标和触摸。
- 器材拖拽、边界限制、碰撞检测、吸附点和连接关系。
- 电路导线端点连接与拓扑校验。
- 物体挂接、放置、浸没、加热、计时、测量和读数。
- 步骤状态机：未开始、可执行、已完成、错误、实验完成。
- 无效操作反馈：说明失败原因和下一步，不直接替学生完成。
- 撤销、重做、重置本组、重置整个实验。
- 多组实验、改变变量和控制变量校验。
- 数据表格、公式计算、单位、有效位数和结论判断。
- 本地会话保存和恢复。

实验控制器不得直接操作页面 DOM。控制器接收实验状态和用户动作，返回新的实验状态、物理读数、步骤结果和反馈消息，便于独立测试。

### 4.4 实验控制器族

65 项实验按物理行为复用控制器基础能力：

- 测量类：刻度尺、停表、温度计、天平、量筒、弹簧测力计、电表。
- 热学类：热量、温升、相变平台、沸腾、扩散和热机能量转换。
- 力学类：运动积分、摩擦、平衡、压强、浮力、机械能和简单机械。
- 光学类：光线追迹、反射、折射、透镜成像和光屏承接。
- 声学类：振动、频率、振幅、介质和传播效果。
- 电路类：节点、支路、短路/断路检测、欧姆定律、电功率和焦耳热。
- 电磁类：磁场方向、线圈匝数、电流、受力和电磁感应。
- 模型类：安全模拟链式反应、电磁波传播和危险实验替代。

共享的是交互基础设施和仪器模型，不强迫不同物理规律使用同一个万能公式引擎。

## 5. 交互标准

参考浮力实验的核心标准应用到所有实验：

1. 学生必须亲自操作器材，不能只点击“播放动画”。
2. 器材靠近正确位置时吸附，错误位置不吸附并给出原因。
3. 步骤由真实实验状态解锁，不允许跳过必要测量直接完成。
4. 仪器读数由当前物理状态计算，不写死为展示数字。
5. 至少完成一组有效数据；需要控制变量或验证普遍规律的实验必须支持多组数据。
6. 数据记录后自动进入表格，派生量由公式计算并显示单位。
7. 系统根据容差判断数据是否支持目标规律，偏差过大时指出可能操作原因。
8. 重置后物理状态、器材位置、步骤和本组数据一致恢复。
9. 所有关键动作都可撤销，避免一次误操作迫使学生重做全部实验。
10. 危险实验使用安全的虚拟过程，不鼓励现实中复现危险操作。

### 5.1 每个实验的完成验收

一个实验只有同时满足以下条件才算完成：

- 六项教学内容齐全且与工作簿一致。
- 所有清单器材都可识别，关键器材可交互。
- 从初始状态能够通过用户操作走到完成状态。
- 至少存在一条错误操作路径，并给出准确反馈。
- 物理计算有单元测试，单位和有效位数正确。
- 至少记录一项有效观测或测量；定量实验记录原始量和派生量。
- 可重置并重新完成，不残留上一轮状态。
- 会话刷新后可恢复，或可明确选择重新开始。
- 实验报告显示操作记录、数据表、计算、结论和误差/补充说明。
- 桌面端和移动端无文字遮挡、控件重叠或不可操作区域。

仅有参数滑块、自动动画或静态示意图，不满足完成标准。

## 6. 数据记录与报告

会话保存在浏览器 `localStorage`，不依赖登录或后端，保证 GitHub Pages 和本地预览均可使用。会话结构包括：

- 实验 ID、会话 ID、开始和更新时间、完成状态。
- 每一步操作事件和反馈结果。
- 每次测量的原始值、单位、时间和实验条件。
- 公式计算得到的派生值。
- 多组实验记录。
- 自动结论、误差提示和实验补充。

报告固定包含：实验目的、实验器材、操作记录、原始数据、计算结果、实验结论、误差分析和实验补充。学生可以查看历史会话并继续未完成实验。

本地存储读取失败、数据版本过旧或内容损坏时，系统保留原数据并提示新建会话，不让实验页面白屏。

## 7. 视觉与可用性

- 教材列表采用图一的课程目录密度和信息层级，但沿用“教育之美”现有导航和品牌。
- 实验台采用图二的深灰工作区和真实器材感，信息阅读区保持现有浅色界面。
- 关键物理量使用不同颜色，但颜色不是唯一状态提示，同时配合文字、图标或线型。
- 图标按钮提供可访问名称和悬停提示。
- 实验场景使用稳定宽高比和响应式约束，器材读数变化不得引发布局跳动。
- 桌面端优先适配 1366×768、1440×900 和 1920×1080；移动端优先适配 390×844。
- 交互命中区域不小于 40×40 CSS 像素，支持键盘访问非拖拽命令。

## 8. 错误处理与安全

- 短路、反接、超量程、未校零、未断开开关改线等操作给出实验相关反馈。
- 超出物理模型有效范围的输入被限制，并解释限制原因。
- 明火、市电、高压、有毒物质、汞、碘蒸气和放射性过程只做安全虚拟模拟。
- 页面运行异常由实验级错误边界捕获，保留当前会话并允许重新加载实验。
- 单个实验加载失败不影响教材目录和其他实验。

## 9. 测试与验证

### 9.1 自动化测试

- 数据完整性：65 项、册别数量、章节、必做/选做和六项内容完整。
- 控制器单元测试：物理公式、边界条件、单位换算和容差判断。
- 状态机测试：正确路径、错误路径、步骤顺序、重置、撤销和重复实验。
- 会话测试：保存、恢复、版本迁移和损坏数据回退。
- 路由测试：教材实验与拓展实验互不冲突，GitHub Pages 基础路径正确。

### 9.2 浏览器验证

- 对每个实验至少执行一次完整的真实交互流程。
- 桌面和移动视口截图检查列表、详情、实验台和报告。
- 对拖拽实验执行关键器材移动、吸附、读数和记录断言。
- 检查控制台错误、空白场景、资源 404、文字遮挡和滚动锁死。

测试通过不等于实验完成；必须以真实交互流程能够产生有效实验数据作为最终证据。

## 11. 非目标

- 第一版不建设账号、班级、云端作业分发和教师管理后台。
- 第一版不复刻参考 HTML 中仅用于演示的“推送学生端”和“分享实验”假按钮。
- 不把全部实验强制制作成 3D；采用更适合教学、性能和触控操作的 2D SVG/Canvas 仿真。
- 不重写现有 133 个拓展实验的物理模型，只调整其目录归属并保持现有入口可用。

## 12. 成功标准

项目完成时必须同时满足：

- 教材实验目录准确展示 65 项，并可按版本、册别、章节和必做/选做筛选。
- 九年级 26 项、八年级下册 18 项、八年级上册 21 项全部可交互完成。
- 每项实验都有六项教学内容、有效数据记录和实验报告。
- 现有 133 个实验仍可从“拓展实验”访问。
- 本地开发服务器支持实时预览。
- GitHub Pages 提供与本地验收一致的最新生产版本。
